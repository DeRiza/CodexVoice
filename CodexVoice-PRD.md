## 2. 产品目标

CodexVoice 是一个 macOS 本地语音输入工具：

用户在任意 App 的输入框中按下全局热键，说话，再按一次热键结束。应用在本地完成语音转文字，并把文字稳定注入当前光标位置。

第一阶段只追求可靠闭环，不追求复杂智能能力。

---

## 3. 成功标准

MVP 成功的定义：

- 用户可以从源码启动应用。
- 菜单栏能显示应用状态，并可退出。
- 全局热键在常见 App 中可触发录音。
- 默认麦克风能录到音频。
- 停止录音后，本地模型能返回中文文本。
- 文本能稳定注入 Notes、TextEdit、浏览器输入框、聊天输入框等常见目标。
- 音频不上传云端，不写入临时音频文件。
- 权限缺失、模型缺失、麦克风不可用时有明确错误提示。

---

## 4. 用户场景

### 4.1 短句输入

用户在聊天软件、浏览器输入框或笔记应用中快速输入一句中文。

期望体验：

1. 点击目标输入框。
2. 按 `Cmd+Shift+Space`。
3. 说一句话。
4. 再按 `Cmd+Shift+Space`。
5. 文本出现在光标位置。

### 4.2 中等长度段落

用户在文档或笔记软件中输入一段 10-30 秒的内容。

期望体验：

- 录音期间浮窗提示正在录音。
- 停止后进入处理中状态。
- 完成后一次性注入最终文本。
- 单次最长录音默认 60 秒。

### 4.3 权限或环境异常

用户首次运行时未授权麦克风或辅助功能。

期望体验：

- 应用明确提示缺少哪项权限。
- 不崩溃、不静默失败。
- 给出系统设置入口或操作说明。

## 5. MVP 范围

### 5.1 应用生命周期

- macOS 菜单栏应用。
- 启动、退出、状态显示。
- 单实例运行，避免多个实例同时监听热键。
- 基础日志，默认写入用户目录下的应用日志路径。

### 5.2 热键

- 默认热键：`Cmd+Shift+Space`。
- 行为：toggle。
- 第一次触发：开始录音。
- 第二次触发：停止录音并进入转录。
- MVP 不支持 Fn 键。
- MVP 不实现“按住说话、松开结束”。

### 5.3 录音

- 默认麦克风。
- 采样率：16kHz。
- 声道：mono。
- 格式：16-bit PCM。
- 录音数据只在内存中保存。
- 不生成临时 wav/mp3/m4a 文件。

### 5.4 停止规则

MVP 只保留可解释、可测试的规则：

| 规则 | 默认值 | 说明 |
|------|--------|------|
| 手动停止 | 再按热键 | 最高优先级 |
| 说话前无输入超时 | 8 秒 | 防止误触后无限等待 |
| 说话后静音停止 | 1.2 秒 | 可配置范围 0.8-3.0 秒 |
| 单次录音硬上限 | 60 秒 | 防止长时间占用麦克风 |

旧文档中的 800ms、2s、30s 多套静音规则不再并存。30s 长等待只作为未来“长听写模式”候选，不属于 MVP 默认行为。

### 5.5 本地转录

- 使用本地 Whisper 系列模型。
- 默认中文优先。
- 模型选择必须经过一次技术 spike：
  - 首选方向：Apple Silicon 上评估 MLX Whisper。
  - 备选方向：faster-whisper 作为稳定基线。
- 不承诺 Apple ANE 加速；除非后续有可验证实现。
- 转录接口必须接受内存 PCM，不允许为了模型调用写临时音频文件。

### 5.6 文本注入

- MVP 默认使用剪贴板粘贴并恢复用户原剪贴板。
- 原因：对中文、多 App、富文本输入框更稳定，代码量更少。
- CGEvent 逐字 Unicode 注入不作为 MVP 默认路径。
- 注入失败时必须返回错误并提示用户，不允许假装成功。

### 5.7 UI

- 菜单栏状态项。
- 简洁浮窗，至少支持四种状态：
  - idle
  - recording
  - processing
  - error
- MVP 不做复杂设置面板。
- 可通过配置文件调整基础参数。

### 5.8 配置

MVP 需要最小配置：

```yaml
hotkey: "cmd+shift+space"
recording:
  sample_rate: 16000
  channels: 1
  pre_speech_timeout_sec: 8.0
  post_speech_silence_sec: 1.2
  max_duration_sec: 60.0
transcription:
  engine: "auto"
  language: "zh"
injection:
  method: "clipboard"
ui:
  show_overlay: true
logging:
  level: "INFO"
```

实现时可以补充字段，但不得引入与 MVP 无关的复杂配置。

## 6. MVP 不做项

以下功能明确不进入 MVP，不得在 README 中写成已有功能：

- Qwen 0.5B 标点归化。
- 个人词库、jieba、SQLite、TF-IDF。
- 流式增量注入。
- 录音中手动编辑不中断。
- 语义断句重评分。
- 多页设置面板。
- 模型下载管理器。
- 多设备选择 UI。
- LaunchAgent 后台常驻。
- OpenAI API 代理、证书、PF、hosts 修改。
- “听写完全对标 macOS 系统听写”的承诺。

## 8. 技术原则

### 8.1 macOS 优先

- 使用 macOS 原生菜单栏、权限、粘贴板和事件能力。
- 避免为了 UI 引入大型跨平台框架，除非原生方案阻塞。

### 8.2 本地优先

- 音频不上传。
- 音频不落盘。
- 模型缓存可以落盘，但必须位于清晰的用户缓存目录。

### 8.3 稳定优先

- 先实现 batch 转录，再考虑 streaming。
- 先一次性注入最终文本，再考虑增量注入。
- 先配置文件，再考虑复杂设置面板。

### 8.4 可验证优先

每个功能必须能被手动验收或自动测试证明。不能验证的能力不进入 MVP。

## 9. 验收清单

### 9.1 功能验收

- 启动应用后菜单栏出现状态项。
- 热键能开始录音。
- 再按热键能停止录音。
- 空录音或静音录音不会注入乱码。
- 中文短句能完成转录和注入。
- 注入完成后用户原剪贴板被恢复。
- 权限缺失时提示明确。

### 9.2 稳定性验收

- 连续执行 20 次短句输入，不重复注入、不丢焦点。
- 麦克风不可用时不崩溃。
- 模型不可用时不崩溃。
- 目标 App 不接受粘贴时能报告失败。
- 单次录音超过硬上限后自动停止。

### 9.3 隐私验收

- 转录流程不创建临时音频文件。
- 日志不记录完整音频内容。
- 默认不上传音频和转录文本。

## 10. 关键风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| MLX Whisper 集成不稳定 | 阻塞转录 | 先做 spike；失败则切 faster-whisper |
| 剪贴板恢复时序不稳定 | 影响用户剪贴板 | 注入前保存，粘贴后延迟恢复，并保留失败日志 |
| 全局热键权限不完整 | 热键无响应 | 启动时检查辅助功能权限 |
| 不同 App 粘贴行为不同 | 注入失败或格式异常 | 验收覆盖 Notes、TextEdit、Chrome/Safari、聊天输入框 |
| 旧文档功能回流 | 范围失控 | README/PRD 明确不做项，PM gate 控制 |
