# Streaming Transcription Design

> 状态: 后续阶段设计，不属于 v3.0 MVP  
> 日期: 2026-06-22

---

## 1. 结论

流式能力暂不进入 MVP。

MVP 只做 batch 流程：

```
录音结束 -> 本地转录 -> 一次性注入最终文本
```

原因：

- 流式转录会引入 chunk、重叠窗口、去重、顺序控制和 UI 同步。
- 流式增量注入很容易造成重复文本、漏字、光标错位和用户编辑冲突。
- Whisper 类模型通常不是原生增量转录模型，模拟流式需要额外复杂度。
- 第一阶段的核心风险是“能否稳定注入文本”，不是“能否边说边出字”。

因此，流式能力必须等 MVP 验收通过后再评估。

---

## 2. 未来目标

后续阶段如果实现流式，目标顺序必须是：

1. **流式预览**：录音时在浮窗中显示临时文本。
2. **最终注入**：停止后仍只注入一次最终文本。
3. **增量注入实验**：只有预览稳定、去重稳定、跨 App 粘贴稳定后才评估。

禁止直接从 MVP 跳到“边说边往目标 App 注入”。

---

## 3. 为什么不做 MVP 流式增量注入

### 3.1 Whisper 模型限制

Whisper 系列模型更适合整段音频转录。所谓“流式”通常是：

```
音频切 chunk -> 每个 chunk 单独转录 -> 文本合并 -> 去重
```

这不是真正的 encoder 状态复用。chunk 太短会影响准确率，chunk 太长会增加延迟。

### 3.2 注入风险

增量注入到前台 App 有几个难点：

- chunk 重叠导致重复字。
- 模型后续可能改写前文。
- 用户可能在转录过程中移动光标或编辑文本。
- 不同 App 对粘贴、选区、撤销栈的处理不同。

这些问题会直接破坏用户正在输入的内容，所以不能作为第一版能力。

### 3.3 代码复杂度

流式需要新增：

- ring buffer。
- chunk scheduler。
- 推理队列。
- 文本合并器。
- 去重算法。
- 预览状态。
- 注入回滚策略。

这与 v3.0 “代码量少、稳定优先”的原则冲突。

## 4. 推荐的后续架构

### 4.1 Phase 4A - 流式预览

```
Recorder
  -> ChunkBuffer
  -> StreamingPreviewTranscriber
  -> PreviewMerger
  -> Overlay preview text
```

特点：

- 只更新浮窗预览。
- 不写入目标 App。
- 停止录音后重新做一次最终转录，保证注入文本质量。

### 4.2 Phase 4B - 最终文本优化

```
chunk preview text
  -> user sees progress
final full audio
  -> full transcription
  -> one-shot injection
```

特点：

- 预览只服务“心理等待时间”。
- 最终文本仍来自完整音频，避免 chunk 边界误差。

### 4.3 Phase 4C - 增量注入实验

只有满足以下条件才允许进入：

- batch MVP 已稳定。
- 流式预览连续 50 次无重复/漏字严重问题。
- 至少 5 个常见 App 的粘贴行为已验证。
- 有明确撤销或替换策略。

## 5. Chunk 策略候选

如果后续实现流式预览，可从以下参数开始：

| 参数 | 候选值 | 说明 |
|------|--------|------|
| chunk_duration_sec | 2.0 | 每次新增音频长度 |
| overlap_sec | 0.5 | 上下文重叠 |
| max_pending_chunks | 1 | 保持顺序，避免并发乱序 |
| language | zh | 中文优先时跳过语言检测 |

这些参数只用于预览，不影响 MVP。

---

## 6. 文本合并原则

预览合并只能影响浮窗，不能影响目标 App 内容。

候选策略：

- 前缀/后缀最长公共串去重。
- 时间戳去重。
- 保留最近 N 秒文本作为上下文。
- 当模型改写前文时，直接替换浮窗预览，不尝试修改目标 App。

---

## 7. 验收标准

流式预览阶段的验收：

- 录音 10 秒以上时，浮窗能出现预览文本。
- 停止后最终注入文本不重复、不依赖预览文本。
- 预览失败不影响最终转录和注入。
- 关闭流式预览后，MVP batch 路径完全不受影响。

增量注入阶段的验收：

- 连续 100 次短句输入不重复注入。
- 用户在目标 App 中移动光标时能安全停止或放弃增量注入。
- 支持撤销语义，不破坏目标 App 原文本。

---

## 8. 明确禁止

在 MVP 阶段禁止：

- 创建 `StreamingTranscriber` 作为主流程依赖。
- 为流式预览改动注入逻辑。
- 把 preview 文本直接注入目标 App。
- 为了流式转录写临时音频文件。
- 在 README 中声明“边说边出字”为已实现功能。
